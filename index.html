Reinforcement Learning & MDP - Review Questions

Q1. What is the typical convergence rate of gradient descent on a smooth and strongly convex function?
  • Answer: Linear O(ρ^t).
  • Analysis: For a function that is both smooth and strongly convex, the gradient cannot be small unless it is already near the optimum, resulting in a linear convergence rate. By contrast, if a function is only convex, the convergence rate is sublinear O(1/t).
Q2. How is the "Markov Property" formally defined in the context of Reinforcement Learning?
  • Answer: A state S_t is Markov if and only if P[S_{t+1}|S_t] = P[S_{t+1}|S_1,..,S_t].
  • Analysis: The Markov Property dictates that "the future is independent of the past given the present". Because the state captures all relevant information from the history, it acts as a sufficient statistic of the future.
Q3. What is the fundamental assumption about goals in Reinforcement Learning, known as the "Reward Hypothesis"?
  • Answer: All goals can be described by the maximization of expected cumulative reward.
  • Analysis: In the RL framework, the agent receives a scalar feedback signal called a reward. The agent's overarching job is to select actions to maximize the total future cumulative reward.
Q4. What is the difference between a Fully Observable Environment and a Partially Observable Environment (POMDP)?
  • Answer: In full observability, the agent directly observes the environment state. In partial observability, the agent only indirectly observes the environment and must construct its own state representation.
  • Analysis: In a fully observable MDP, the agent state equals the environment state. In a POMDP, the agent might use a complete history or a recurrent neural network to construct its state representation.
Q5. What are the three major components that might make up a Reinforcement Learning agent?
  • Answer: A Policy, a Value Function, and a Model.
  • Analysis: The policy is the agent's behavior function mapping states to actions. The value function evaluates how good a specific state or action is. The model is the agent's internal representation of the environment.
Q6. How does Reinforcement Learning fundamentally differ from Planning?
  • Answer: In RL, the environment is initially unknown, and the agent learns via interaction. In Planning, a perfect model of the environment is known, and the agent computes its policy internally.
  • Analysis: Planning is akin to deliberation or search, where an agent can query an internal emulator. RL requires interactive trial-and-error gameplay.
Q7. Why must an agent balance Exploration and Exploitation?
  • Answer: Exploration discovers more information about the environment, while exploitation leverages known information to maximize immediate rewards.
  • Analysis: Over-exploiting might leave the agent stuck in a suboptimal strategy. Both are necessary to find the best long-term strategy.
Q8. How does a Value-Based RL agent differ from a Policy-Based RL agent?
  • Answer: A value-based agent estimates the value of a state/action and picks the highest-valued option. A policy-based agent directly learns a mapping from states to actions.
  • Analysis: Value-based methods (like Q-Learning) handle discrete action spaces. Policy-based methods are suited for continuous action spaces. Actor-Critic methods combine both.
Q9. What does it mean for an RL algorithm to be "Model-Free"?
  • Answer: It treats the environment as a black box and learns optimal behavior purely through trial and error without learning the environment's transition and reward functions.
  • Analysis: This is compared to model-based methods, where the agent learns or is given the transition function to simulate future trajectories.
Q10. What is the fundamental difference between On-Policy and Off-Policy learning?
  • Answer: In on-policy learning, the target policy being optimized is the same as the behavior policy. In off-policy learning, the target policy is different from the behavior policy.
  • Analysis: Off-policy methods have higher data efficiency because they can use an Experience Replay Buffer to reuse past transitions.
Q11. What are the key advantages and drawbacks of Q-Learning?
  • Answer: Pros: efficient on-line learning and off-policy learning via exploration. Cons: overestimation bias making the agent overly optimistic.
  • Analysis: Q-Learning is a foundational value-based, model-free, and off-policy algorithm that updates action-values using the temporal difference formula.
Q12. In what situations might SARSA be preferred over Q-Learning?
  • Answer: SARSA is preferred in hazardous environments or when dealing with constraints because it learns from the actual path taken.
  • Analysis: Because SARSA is an on-policy method, it is safer but generally slower to reach an optimal policy than Q-Learning.
Q13. How does Deep Q-Network (DQN) solve problems in high-dimensional state spaces?
  • Answer: DQN uses deep neural networks to handle high-dimensional states (like pixels) and breaks data correlation using Experience Replay.
  • Analysis: While DQN successfully combines deep learning with Q-learning, it remains limited to discrete action spaces and is prone to overestimation bias.
Q14. Why is Proximal Policy Optimization (PPO) often considered an "industry standard"?
  • Answer: PPO is popular due to its stable and reliable convergence. It prevents destructively large updates by constraining how far the new policy can deviate.
  • Analysis: PPO is an on-policy, model-free Actor-Critic algorithm. Despite its stability, it suffers from lower sample efficiency compared to off-policy methods.
Q15. What are the distinct benefits of Soft Actor-Critic (SAC)?
  • Answer: SAC is highly sample-efficient and utilizes entropy regularization for superior exploration, making it SOTA for continuous control.
  • Analysis: SAC is a model-free, off-policy, Actor-Critic architecture. It has the drawback of being computationally intensive during training.
